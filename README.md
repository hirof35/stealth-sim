
# ステルスAI 視界判定シミュレーター (Stealth AI Vision Simulator)

Node.jsのコンソール上で動作する、ゲームの敵AIにおける「視界判定（Line of Sight）」と「状態遷移（発見・捜索・見失う）」のアルゴリズムを可視化するシミュレーターです。

ステルスゲーム（隠れる要素のあるゲーム）の裏側で動いている基本ロジックを、視覚的に理解することができます。

---

## 🚀 機能特徴

- **3段階の視界チェック**: 「距離」「角度（視野角）」「遮蔽物（レイキャストによる壁判定）」を毎フレーム計算。
- **状態遷移（ステートマシン）の再現**: プレイヤーを検知すると追跡（`CHASE`）に、壁に隠れると捜索（`SEARCH`）に切り替わり、時間経過で見失う（`PATROL`に戻る）リアルタイムAIロジック。
- **外部ライブラリ不要**: Node.jsの標準機能（JavaScript）のみで軽量に動作。

---

## 🛠️ 動作環境

- Node.js: v22.0.0 以上推奨（v14以降であれば動作します）
- OS: Windows (PowerShell / コマンドプロンプト), macOS, Linux

## 📦 インストールと実行方法

### 1. プロジェクトフォルダの準備
ターミナル（またはPowerShell）を開き、プロジェクト用のフォルダを作成して移動します。

```bash
mkdir stealth-sim
cd stealth-sim
2. ファイルの作成
フォルダ内に simulation.js という名前でファイルを作成し、シミュレーターのソースコードを貼り付けて保存します。

3. シミュレーションの実行
以下のコマンドを実行すると、コンソール上でリアルタイムアニメーションが開始されます。

Bash
node simulation.js
🔍 画面の見方とマップ記号
実行すると、1秒ごとにコンソール画面が更新され、キャラクターが自動で動きます。

Ｅ (Enemy) : 敵キャラクター。右方向（0ラジアン）を基準に、前方90度の視野を持っています。

Ｐ (Player) : プレイヤー。自動ルートで移動し、敵に近づいたあと壁の裏へ隠れます。

■ (Wall) : 進入不可・視界を遮る「壁」です。

？ (Last Known Position) : 敵がプレイヤーを「最後に目撃した場所」。捜索状態の時だけ出現します。

． (Floor) : 通路（何もない床）です。

敵のステート（状態）
PATROL (巡回) : プレイヤーが視界に入っていない通常状態。

CHASE (追跡) : プレイヤーを視界内に捉え、発見している状態。

SEARCH (捜索) : プレイヤーが壁の裏などに隠れた直後、最後に見た場所（？）の周囲を警戒している状態（4秒間維持）。

🧠 アルゴリズムの解説（コードの核心部）
このシミュレーターは、以下の3つの視覚アルゴリズムを組み合わせて「見えているか」を判定しています。

距離の判定 (Distance Check)
ピタゴラスの定理を用いて敵とプレイヤーの直線距離を計算し、設定された視認距離（デフォルト: 6マス）以内かを判定します。

視野角の判定 (Cone Check)
ベクトルの内積（Dot Product）を利用。敵の正面向きベクトルと、プレイヤーへの方向ベクトルの角度を算出し、視野角（左右45度ずつ）に収まっているかを一瞬で判定します。

遮蔽物の判定 (Raycasting)
敵からプレイヤーに向けて仮想の光線（レイ）を1マスずつ進め、途中に壁（#）が存在しないかを走査します。これにより「壁の裏に隠れる」というステルスアクションが可能になります。
